# 薄弱点反馈：显示给用户 + 在后续作业中混入复习题

薄弱点（系统在通过判定时记录的、学生尚未掌握的知识点）有两条用途：

1. 在学习报告页直接呈现给用户，支持元认知（V1）。
2. 在后续 KP 的作业生成时，主动混入针对前面薄弱点的复习题（约每 N 道题中 1 道，N 待定）。

薄弱点**不影响**讲解阶段的对话内容。

## Considered Options

- **不影响系统行为，仅显示给用户（N1）**：教学上最保守，放弃了"系统主动帮学生复习"的产品价值。
- **影响后续 KP 的讲解 prompt（N2/N3）**：实施挑战多（薄弱点结构、相关性匹配、prompt 效果验证、数据稀疏期），最致命问题是答辩 demo 不可见 —— 评委看到一段讲解无法判断它是否被薄弱点信息影响过。

## Consequences

- 作业生成 prompt 必须接受额外参数：当前 KP 之前的薄弱点池快照 + 混入题数。
- 复习题在作业 UI 上必须明确标注（如"这是基于你前面的 *TCP 三次握手* 出的复习题"），否则学生会困惑。这一标注同时是答辩演示中直接可见的卖点。
- 薄弱点池的查询接口必须支持按章节树位置过滤（只取当前 KP 之前的薄弱点），避免引用未学到的内容。
- 薄弱点池有两条来源（系统判定 + 用户强行跳过），均混入复习题的候选池，不区分。
