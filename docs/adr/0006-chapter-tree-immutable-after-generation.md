# 章节树生成后不可手动编辑

LLM 生成的章节树（Chapter / Section / KP）一旦确认，用户不能手动增删改节点。如需调整，只能重新上传资料生成新 Course。

## Considered Options

- **支持节点增删改 + 拖拽重排序**：节点编辑 UI、数据回写、进度迁移、章/节聚合状态重算 —— 累计 2-3 天工作量。
- **只允许重命名**：折中方案。但实现复杂度仍接近全功能编辑（同样要触发聚合状态刷新和持久化），收益不成比例。

## Consequences

- LLM 章节树生成质量直接决定 Course 体验。生成 prompt 的 trial-and-error 是不可省略的工作。
- 答辩演示时不会触发"章节树不对，要改"的需求；如有问题用户可现场重新上传。
- 未来若产品化，节点编辑是必加项。
