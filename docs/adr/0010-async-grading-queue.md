# 评分流程：异步队列

作业评分采用异步队列处理。学生提交后立即返回"已提交，AI 老师批改中"状态；评分完成后通过通知或前端轮询展示结果。讲解阶段的对话仍使用 SSE 流式输出。

## Considered Options

- **同步阻塞**：用户点提交 → 转圈 → 等几秒看到完整反馈。实现最简，但用户在等待时被锁住。
- **SSE 流式（评分阶段）**：评分内容流式推送，用户全程看到 AI 老师"思考写"。视觉冲击大、实现成本接近同步。但与"现实老师批改"的教学叙事不一致。

## Consequences

- 多 1-1.5 天工作量：任务队列（Redis + RQ / Celery / Python asyncio 内存队列任选）+ 状态字段（`pending / running / done / failed`）+ 前端轮询或 WebSocket 推送 + 错误重试逻辑。
- 教学叙事上更接近真实师生交互（"老师收作业 → 改作业 → 发反馈"）。
- 演示中需要清晰的"批阅中"UI 状态指示，避免评委误以为系统卡住。
- 应急砍功能时可降级到同步阻塞（移除队列与状态轮询）。
