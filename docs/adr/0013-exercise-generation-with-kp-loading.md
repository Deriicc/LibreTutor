# 作业题生成时机：与 KP 讲解材料一起懒加载

## Status

**Superseded by ADR-0020**（2026-05-14）。"LLM 一次调用同时输出讲解材料 + 作业题"已经拆开：**KPMaterial** 在课程构建后 prewarm（一次调用），**KPExerciseSet** 在评估完成后 tailor（另一次调用，输入包含对话裁剪后的 covered_concepts）。"作业题与讲解对话无关"这一条决策已**反转**——参见 ADR-0019 / ADR-0020。

下方原文保留以记录历史决策。

---

每个 **KnowledgePoint** 的作业题在用户首次进入该 KP 时一并生成 —— LLM 一次调用同时输出"讲解切入点 + N 道作业题"。作业题在讲解开始前就已固定，**不基于讲解过程的对话内容动态生成**。

参见 ADR-0002（KP 内容懒加载）。

## Considered Options

- **章节树生成时一次性预生成所有 KP 的作业题**：80 KP × 5 题 = 400 题，单次输出 10K+ tokens 截断风险高；分次调用又会让上传后等 5-10 分钟，答辩演示风险大。
- **讲解结束、进入作业阶段时即时生成**：作业题可基于讲解对话内容个性化，但讲完后用户还要等几秒才看到作业（UX 不流畅）；个性化效果在演示中不可见，性价比低。

## Consequences

- KP 进入时的 LLM 调用 output schema 必须同时包含讲解材料字段和作业题字段。
- 作业题与讲解阶段的实际对话内容**无关**，缺失"基于本节讲解的个性化"。但薄弱点复习题混入机制（ADR-0005）从另一维度提供个性化，覆盖已足。
- 应急砍功能时无降级路径 —— 1.B 已是最简方案。
