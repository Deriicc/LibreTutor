# 作业题数量配比、复习题混入与存储策略

收口 C 的三个参数级子决策（生成时机见 ADR-0013，生成方法见 ADR-0014，薄弱点反馈见 ADR-0005）。

## 数量与配比

每次作业固定 **5 道题**：

- 存在历史薄弱点时：4 道当前 KP 题（3 MCQ + 1 简答）+ 1 道复习 MCQ
- 无历史薄弱点时：5 道全是当前 KP（3 MCQ + 2 简答）

MCQ 题型从 Pro-QuEST 12 类中选 3 种不同（Definition / Comparison / Causal Consequence / Quantification / Interpretation 等）。

## 复习题混入

- 复习题从"当前 KP 之前的薄弱点池"中随机抽取 1 个薄弱点作为题源。
- 复习题题型固定为 MCQ（评分稳定）。
- 复习题在 UI 上显式标注（参见 ADR-0005）。

## 存储与重做

- 作业题生成后立即持久化（题目 + 学生答案 + AI 评分 + 反馈），按 `KP × 提交序号` 绑定。
- 学生选"再练一道"时由 LLM 重新生成新题，不重用历史题。
- 历史题保留作为学习报告页"过往作业"的展示数据。

## Consequences

- 学生每次作业用时约 5-10 分钟。
- 复习题混入需要"当前 KP 之前的薄弱点池"查询接口（ADR-0005 已要求）。
- 重做生成新题增加 LLM 调用次数；学生反复重做会推高 API 成本，需观测。
- 历史题持久化使学习报告页能展示完整作业轨迹，是答辩演示的核心可视化资产。
