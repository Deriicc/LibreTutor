# 苏格拉底 prompt 架构与核心行为模式

## Prompt 分层架构

苏格拉底对话的 system prompt 由三层拼接组成：

1. **Layer 1（全局基础层 / 硬编码）**：苏格拉底教学法的核心原则与默认行为脚本（见下"核心行为"部分）。**这是整个系统最关键的 prompt 工程产物**。
2. **Layer 2（Course 级 / 用户在 AI 教师设定页可改）**：教学风格、目标、特殊要求（具体可配置项见后续 ADR）。
3. **Layer 3（KP 级 / 进入 KP 时由 LLM 即时生成）**：当前 KP 的讲解切入点、预期 keyphrase、可能的学生错误模式。由 KP 进入时的 LLM 调用产出（参见 ADR-0002、ADR-0013）。

KP 切换时只替换 Layer 3，Layer 1+2 保持。

## 核心行为模式

### 开场策略
进入 KP 的第一句话由 LLM 主动发起：**"你对 <KP 名称> 已经知道什么？"** 或同义变体。先问学生已知（摸底），不直接讲授。

### 学生答错时
**先反问/给提示一次，仍错则直接纠正 + 解释**。
- 第 1 次错 → 反问（"为什么你这么想？"）或给提示（"考虑 <角度> 这一面"）
- 第 2 次错或学生坚持原答案 → 给正确答案 + 解释 + 继续

### 学生答"不知道"或沉默
**拆解成更简单的子问题**。从 KP 核心概念降阶到学生可能掌握的更基础概念，逐步爬升。

### 讲解结束判定
**LLM 自主判断 + 学生主动结束权混合**：
- LLM 判断学生已掌握主要内容时，主动询问："看来你理解了主要内容，要不要进作业试试？"
- 学生在任何时候可主动说"我懂了 / 进作业"，强制结束讲解。

学生强制跳过的情况由 ADR-0004 的"通过判定"接管：进作业 → 评分 → 系统建议 → 用户决定。

### 单次讲解的最大对话长度
**软限制 10-20 轮**。
- 每 5 轮内部判断一次"是否该结束"。
- 达到 20 轮强制询问学生"要不要进作业"。

## Considered Options

- **单一全局 prompt（不分层）**：所有 KP 共用同一段 prompt，无法响应用户的教师设定，呆板。
- **多 agent 架构（导师 agent + 助教 agent）**：架构上更优雅，但 12 天约束下做不出可演示版本。
- **行为模式的纯苏格拉底（只反问、永不纠正）**：学生卡死。
- **行为模式的纯讲授（直接讲）**：失去苏格拉底法的核心价值，无创新点。

## Consequences

- **Layer 1 的 prompt 文本是整个项目最关键的工程产物**。需要预留至少 1-2 天 trial-and-error 时间，用真实 KP 内容反复测试。
- 答辩时"苏格拉底法怎么实现的"问题，可指向 Layer 1 prompt 文本作为具体证据。论文/PPT 中应包含该 prompt 的核心片段。
- Layer 3 由 KP 进入的 LLM 调用同时产出（与讲解材料、作业题在同一次输出中），需在 schema 中保留"socratic_layer3"字段。
- 学生提前结束讲解（B-4.d）的退出路径必须在前端 UI 上明确（一个"我懂了 / 进作业"按钮）。
- "10-20 轮"软限制需要在前端做轮数计数 + 内部状态同步给 LLM。
