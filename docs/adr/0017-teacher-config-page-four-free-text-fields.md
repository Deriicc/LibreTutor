# AI 教师设定页：4 字段自由文本 + Placeholder + 兜底默认

每个 **Course** 拥有一份 AI 教师设定（参见 ADR-0016 中的 Layer 2）。设定页提供 4 个**自由文本**字段：

1. **人物设定**：老师"是谁"。例如：_"你是一位有 20 年教龄的计算机系教授"_
2. **教学风格**：老师"怎么教"。例如：_"严谨细致，注重逻辑推导；学生答错时先反问，给一次引导后再纠正"_
3. **你的背景**：学生自我描述。例如：_"大二计算机系，对计算机网络零基础，目标是期中考试"_
4. **学习目标**：学生希望达成的目标。例如：_"5 月底前掌握 TCP/IP 协议栈的工作原理"_

## Placeholder 示例 + 帮助文案

每个字段下方显示 placeholder 示例文本（即上述例子）。设定页顶部提供短说明：

> 人物设定 = 老师是谁；教学风格 = 老师怎么教。两者互补，建议都填。

## 兜底默认（字段为空时）

字段为空时，Layer 2 prompt 自动插入硬编码兜底：

- **人物设定**未填 → _"你是一位耐心、有教学经验的老师"_
- **教学风格**未填 → _"友好、循循善诱、采用苏格拉底式教学法"_
- **你的背景**未填 → _"学习者背景未提供，请在对话中适时询问"_
- **学习目标**未填 → _"学习目标未提供，请关注学生在对话中表达的需求"_

## Considered Options

- **预设按钮 + 自由文本（原 3.2）**：演示直观但配置空间小，无法表达复杂需求。
- **高度可配置（按钮 + 滑块 + 8+ 选项，原 3.3）**：用户多数用不完，工程量爆炸。
- **完全开放给用户改 Layer 1 苏格拉底核心原则**：会破坏系统的核心教学方法卖点。**拒绝**。

## Consequences

- Layer 2 prompt 拼装逻辑：4 字段拼接 + 空字段替换为兜底 + 整体注入 system prompt。
- "人物设定"与"教学风格"在自由文本下可能语义重叠（学生写出风格矛盾的内容），placeholder + 帮助文案缓解但不消除，是已知潜在失败模式。
- 设定修改影响**仅限新启动的对话**；已有对话历史的 prompt 不回填重写（避免上下文断裂）。
- Layer 1（硬编码的苏格拉底核心原则，ADR-0016）**禁止**用户修改。
