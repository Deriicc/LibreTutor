# 苏格拉底对话的上下文管理与失败模式处理

收口 I 的剩余决策（架构与核心行为见 ADR-0016；设定页见 ADR-0017）。

## 上下文管理

### 单 KP 内对话历史
完整保留至 KP 结束。20 轮 × ~200 tokens ≈ 4K，远低于 LLM 上下文窗口，不需 sliding window。

### KP 切换时
完全清空前一 KP 的对话历史。新 KP 的 system prompt 仅包含 Layer 1 + Layer 2 + 当前 KP 的 Layer 3（参见 ADR-0016）。

### 学习进度注入 Layer 3
进入 KP 时 LLM 调用的 system prompt 中包含进度概览：

> "你正在教第 X 个知识点：<KP 名称>。该学生之前学过：<Y₁>, <Y₂>, ...。接下来还要学：<Z₁>, <Z₂>, ..."

让 LLM 拥有全局视角，避免重复讲解或暗示未来内容。

### 薄弱点不注入讲解 prompt
重申 ADR-0005：薄弱点仅影响后续作业出题（复习题混入），不影响讲解阶段的 Layer 1/2/3 任何一层。

## 失败模式处理

| 失败模式 | 应对 |
|---------|------|
| LLM 偏离苏格拉底法（直接讲授） | Layer 1 prompt 显式禁止 + few-shot 示例 + prompt 调试期人工抽查；无运行时检测 |
| LLM 卡住（重复同一问题） | `temperature` 设为 0.7-0.9 + Layer 1 prompt 加"避免重复同一问题，从不同角度切入" |
| 学生抗拒（"不想学了"/"别问了直接讲"） | Layer 1 prompt 指导 LLM 识别情绪信号 + 主动提供选项（休息 / 直接讲 / 进作业） |
| LLM hallucination（超出 KP 内容） | Layer 1 prompt 强约束"仅基于提供的 KP 内容" + 抽样人工检查 |

## Considered Options

- **跨 KP 保留对话历史**：token 累积爆炸 + LLM 易混淆主题。拒绝。
- **运行时检测 LLM 偏离**：再加一道 LLM 调用判断"上一句是否太直接讲授"，token 翻倍 + 复杂度高。12 天约束下不做。
- **薄弱点注入讲解 prompt**：与 ADR-0005 矛盾。拒绝。

## Consequences

- prompt 调试是 12 天里最关键的工程时间投入，建议预留 1-2 天。
- 失败模式**无运行时保障**，演示前必须用真实 KP 内容预演 5-10 遍。
- 答辩中应主动陈述"LLM 教学应用的失败模式与我们的缓解策略"作为局限性讨论，比假装完美更可信。
