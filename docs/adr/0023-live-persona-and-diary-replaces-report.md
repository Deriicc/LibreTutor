# 授课人格 Live 化 + 教师日记本取代学习报告页

教师人设（Persona）从"课程构建时按 kp_id 冻结进 `KPMaterial.layer2_snapshot`、事后改人设对已建课程几乎无效"，改为**完全 live 求值、不快照**：每一回合对话与每一篇日记都取产生那一刻的当前 `TeacherConfig`，保存修改后下一回合即换人（**接受对话中途换人**——视为"换了老师"的可见证明，非 bug）。已产生的产物（已发送消息、已写日记）不回溯改写。`KPMaterial.layer2_snapshot` 机制与 `_build_layer2` 的 snapshot 分支随之废除。

同时新增 **TeacherDiaryEntry**（教师日记，详见 CONTEXT.md）：在 KP 结束（`advance` 的 next/retry）时由当下 live Persona 对该次 `(kp_id, attempt)` 写一篇带署名的私人日记体复盘。**这本可翻页的日记完全取代学习报告页**——不再有任何结构化仪表盘/列表 UI，进度/薄弱点/`Grade`/`KPAssessment` 仅作为日记 LLM 的输入，由老师在散文中带出。

## Considered Options

- **保留 ADR-0020 的全程冻结**：对话与日记自洽，但"修改人设"功能对已建课程几乎是 no-op，产品上形同虚设。否决。
- **仅日记 live、对话仍冻结**：同一 KP 出现"对话是老师 A、日记是老师 B"的割裂，比"改了没反应"更像 bug。否决。
- **per-(KP,attempt) 进入时冻结**（折中）：改人设对未开始的 KP/attempt 生效、进行中的不变。技术上更稳，但用户明确要"保存后下一回合就换"，且接受对话中途换人作为"换老师"的可见证据。否决。
- **日记本与报告页并存（双 tab）**：违背"整合在一起"的诉求。否决。
- **日记保留结构化成绩便签/封面统计**：用户要"纯一本日记、无仪表盘"。否决。

## Consequences

- ADR-0020 的本意"不在进行中对话里换人"被**主动放弃**——这是用户的产品选择（可见的人格切换 = 功能而非缺陷），后来人看到对话中途换人格属预期行为，勿当 bug "修复"。
- ADR-0007 的"错题/薄弱点统一在学习报告页结构化展示"在**展示形态上被取代**（改为日记散文带出）；但其"数据仍记录、仍通过薄弱点具象呈现"的数据契约**仍然成立**——`Weakness`/`Grade`/`KPAssessment` 照常写库，只是降级为日记 LLM 的输入。
- `report.py` 不删，从"`/report` 端点的聚合器"降级为"喂日记生成的聚合层"；`ReportPage.tsx` 与 `/api/courses/{id}/report` 路由移除，入口改为打开日记本。
- 取消快照后，进行中的会话理论上可在两回合间换人格——这是已接受的代价。
- 日记按写作时间编年排序；retry 各成一篇，旧篇连署名固化不回改；同一本日记可有多位作者（人设随时间变化）。
